1주 차에서는 git 과 git허브에 대한 기초적인 지식을 배웠다
git은 개발자들이 코드를 짜는 것을 굉장히 편리하게 만들어주는 도구라고 이해를 했다
웹 개발 종합반 수업을 듣고 숙제를 하면서 코드를 살짝만 바꿔도 실행이 안될 때가 많았다
그럴 때 마다 내가 어디를 어떻게 바꿨었는지 일일이 찾아서 다시 수정을 해야 했고
찾지 못 했을 때는 그냥 싹 갈아 엎고 처음부터 다시 했었는데
git은 저장을하고 commit을 하면 히스토리에 나의 이전 내역들이 다 남으니까
어디서 어떤 부분을 내가 고쳐서 작동이 안되는 지 쉽게 알 수 있고
또, 작동이 되는 부분부터 다시 시작을 할 수 있으니 굉장히 편리할 것이라 생각이 된다

github는 git을 저장하고 다시 불러 올 수 있게 만들어 주는 cloud서비스 같은 것이라고 이해했다
그래서 github에 내 작업을 올리게 되면 다른 사람들과 공유 할 수 있고
내 작업물을 발전 시킬 수 있는 여지가 보이면 github에서 다른 사람이 바로 수정을 할 수도 있고
그 수정본이 나의 마음에 들면 나는 그것을 pull해 와서 내 local repo에 적용을 시킬 수도 있다
git은 개발자 개인을 위해서 만들어진 편리한 도구의 느낌이라면
github는 개발자 pool 전체의 발전을 위해서 만들어진 커뮤니티라는 생각이 들었다



키워드로 넘어가서
git은 commit 잘 하고, github와 연결 잘 시키고, 연결이 잘 되었다면 push 잘 해서 나의 프로젝트
들을 잘 저장하고 버전 정보를 남기는 것이 중요하다고 생각이 되고
다른 개발자들의 유용한 code가 보이면 pull 잘 해와서 적재적소에 사용하는 것이 중요하다고 생
각이 된다

내 컴퓨터에 프로젝트 만들기 > git과 연결 시키기 > 연결된 local repo를 원격repo에 올리기 >
원격 repo의 정보들 pull 해 오기 > 수정 사항 commit하기 > 다시 원격 repo에 push 하기
